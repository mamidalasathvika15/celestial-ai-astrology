# Project Reflection

## What Was Easy

Some parts of development felt straightforward, especially rapid prototyping and experimenting with ideas using AI assistance. Building UI components and iterating on features became faster because changes could be tested quickly.

The modular structure of the project also made it easier to separate features such as the Daily Horoscope, Oracle Chat, Compatibility, and Journal system.

---

## What Was Difficult

The most challenging part was integrating Gemini AI in a way that felt personalized rather than generic.

Other challenges included:

- Debugging runtime and dependency issues
- Managing API integration and prompts
- Handling dynamic user-driven responses
- Improving application structure and readability during iteration

Ensuring the application felt polished and consistent across different features also required multiple refinements.

---

## What I Would Change or Improve

Given more time, I would improve:

- Better personalization and contextual memory
- More polished UI animations and responsiveness
- Improved testing and error handling
- Expanded astrological insights and deeper birth-chart interpretation
- Better optimization and performance

I would also continue refining prompts and response quality to create an even more immersive and meaningful user experience.

---

## Final Thoughts

This project helped strengthen my understanding of frontend development, API integration, debugging workflows, and AI-assisted development. It also highlighted the importance of iteration, experimentation, and balancing technical implementation with user experience.
